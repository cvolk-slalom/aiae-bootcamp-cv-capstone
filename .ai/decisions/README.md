# Architecture Decision Records (ADRs)

> One file per non-trivial decision. Append-only. Never edit an accepted ADR — supersede it with a new one.

## When to write an ADR
Write one if the decision affects any of: architecture, dependencies, conventions, silo boundaries, data model, deployment, or AI-SDLC workflow itself.

If you're unsure, write one. They're cheap.

## File naming
`NNNN-kebab-title.md` where `NNNN` is a zero-padded sequence number (e.g., `0002-pick-frontend-framework.md`).

## Template
See [template.md](template.md). Copy it, increment the number, fill in.
