**Why:** User wants to build a demo product in as few prompts as possible. Goal: a fresh chat with no history can pick up work after reading `AGENTS.md` + 1–2 silo files. System must be lightweight, silo-scoped, and self-maintaining.

**Key decisions made in this session (all rolled into ADR 0001):**
- Silo-first context loading (FE / BE / shared) instead of monolithic instructions.
- ADRs are mandatory for non-trivial choices; cheap to write, append-only, superseded not edited.
- Requirements stay lightweight: single-page backlog + ≤30-line briefs only for non-trivial features.
- Journal entries are bullet-only, appended per session (this kickoff entry is intentionally longer because it's the inventory of the scaffold itself).
- Workflows (`new-feature`, `context-discovery`, `record-decision`, `update-journal`) are the executable recipes that keep the system from rotting.
- No clarifying questions from the AI if `.ai/` already answers it; if it doesn't, the answer gets written back in the same change.
- Stack intentionally left TBD — first product chat picks it via ADR 0002 so we're not pre-committed.
