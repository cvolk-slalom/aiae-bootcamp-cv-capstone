**Why:** User wants to build a demo product in as few prompts as possible. Goal: a fresh chat with no history can pick up work after reading `AGENTS.md` + 1–2 silo files. System must be lightweight, silo-scoped, and self-maintaining.

**AI leverage:**
- **Routing-first design** — `AGENTS.md` is a single ≈50-line file that orients any agent in one read. Mirrored to `.github/copilot-instructions.md` so Copilot Chat, Claude, and Codex all converge on the same entry point.
- **Silo split = token control** — frontend / backend / shared each get their own context file so a future feature chat loads ~1 silo instead of the whole repo. Designed against the failure mode of "agent reads everything and runs out of budget."
- **Compression by default** — every doc capped (≤ 150 lines for context, ≤ 30 lines for briefs, bullets-only for journals). Forces the agent to retrieve, not re-derive.
- **Workflows as executable recipes** — `new-feature.md`, `context-discovery.md`, `record-decision.md`, `update-journal.md` mean the next chat doesn't have to invent process; it follows steps.
- **Anti-clarifying-question rule** — hard rule in `AGENTS.md` that the agent answers from `.ai/` or writes the answer back. Eliminates the most common token sink (back-and-forth Q&A).

**Patterns / learnings:**
- Treat the meta-layer as a *product surface for agents*: every file is consumed by an LLM, so it must be discoverable from one entry point and chunked for partial loads.
- ADRs are cheap insurance — even tiny scaffolding choices got one (ADR 0001) so future chats don't relitigate them.

**Did:**
- Added [scripts/ai-commit.sh](../../scripts/ai-commit.sh) — stages all, commits with AI-author trailer, pushes (sets upstream first time).

- Wrote [ADR 0002](../decisions/0002-auto-commit-on-completion.md); updated ADR index.

**Why:** Single-driver AI-built repo — manual commits create drift between journal/ADRs and git history. Convention makes commits a routine end-step of every task.

**AI leverage:**
- **Automation removed a human-in-the-loop step** — `scripts/ai-commit.sh` is one line for the agent to call; no message-formatting cognitive load. Standard `type: subject` + AI-author trailer means provenance is grep-able in `git log`.
- **Wired into the workflow, not just the docs** — added as step 10 of `new-feature.md` so the next chat doesn't need to remember; it just follows the recipe.
- **Dogfooded immediately** — the change that added the helper was itself shipped via the helper. Validates the loop before any feature work depends on it.
- **Failure modes documented, not handled** — push rejected = stop and report (no force-push escape hatch). Prevents the AI from "helpfully" destroying history.

**Patterns / learnings:**
- Any new convention should ship with: (1) the automation, (2) the workflow doc, (3) the hard rule in `AGENTS.md`, (4) the ADR. Skipping any of the four means it rots.
- Dogfood on the same commit it's introduced — it's the cheapest possible integration test.
